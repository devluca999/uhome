# Production Environment Variables Checklist — uhome

Use this checklist when configuring production and staging environments.

## Frontend (Hosting Platform)

Set in Netlify, Vercel, Cloudflare Pages, etc.

| Variable | Required | Description |
|----------|----------|-------------|
| `VITE_SUPABASE_URL` | Yes | Supabase project URL (e.g. `https://xxx.supabase.co`) |
| `VITE_SUPABASE_ANON_KEY` | Yes | Supabase anonymous/public key |

## Supabase Edge Functions

Configure in Supabase Dashboard → Project Settings → Edge Functions, or via `supabase secrets set`.

| Variable | Required | Used By | Description |
|----------|----------|---------|-------------|
| `SUPABASE_URL` | Yes | All functions | Supabase project URL (auto-injected by Supabase) |
| `SUPABASE_SERVICE_ROLE_KEY` | Yes | Most functions | Service role key for server-side operations |
| `SUPABASE_ANON_KEY` | Yes | rate-limit-*, abuse-guard | Anon key for client-like operations |
| `STRIPE_SECRET_KEY` | Yes | create-connect-account, create-payment-intent, stripe-connect-webhook | Stripe secret key (live for prod, test for staging) |
| `STRIPE_WEBHOOK_SECRET` | Yes | stripe-connect-webhook | Webhook signing secret from Stripe Dashboard |
| `POSTAL_WEBHOOK_SECRET` | Yes | email-webhook | Secret for verifying Postal webhook requests |
| `POSTAL_SMTP_HOST` | Yes | send-email | Postal SMTP host |
| `POSTAL_SMTP_USER` | Yes | send-email | Postal SMTP user |
| `POSTAL_SMTP_PASS` | Yes | send-email | Postal SMTP password |
| `POSTAL_SMTP_PORT` | No | send-email | Default: 587 |
| `POSTAL_API_URL` | No | send-email | Override API URL if different from host |
| `POSTAL_API_KEY` | No | send-email | Fallback to POSTAL_SMTP_PASS if not set |
| `POSTAL_FROM_EMAIL` | No | send-email | Default: noreply@uhome.app |
| `VAPID_PUBLIC_KEY` | Yes | send-push | Web Push VAPID public key |
| `VAPID_PRIVATE_KEY` | Yes | send-push | Web Push VAPID private key |

## GitHub Secrets (CI/CD)

Required for deploy workflows and E2E in CI.

| Secret | Required For | Description |
|-------|--------------|-------------|
| `VITE_SUPABASE_URL` | Production deploy, CI build | Production Supabase URL |
| `VITE_SUPABASE_ANON_KEY` | Production deploy, CI build | Production anon key |
| `VITE_SUPABASE_STAGING_URL` | Staging deploy, CI E2E | Staging Supabase URL |
| `VITE_SUPABASE_STAGING_ANON_KEY` | Staging deploy, CI E2E | Staging anon key |
| `SUPABASE_SERVICE_ROLE_KEY` | Deploy (record-deployment) | Service role for edge functions |
| `VISUAL_TEST_BASE_URL` | Visual tests | Optional; defaults to localhost |

## Storage Buckets

Create in Supabase Dashboard → Storage. Migrations referenced:

- `supabase/migrations/create_storage_buckets.sql` — creates `documents`, `images`, `avatars`
- `supabase/migrations/storage_rls_documents_avatars.sql` — RLS for documents and avatars
- `supabase/migrations/storage_rls_images.sql` — RLS for images
- `supabase/storage-setup.md` — manual setup notes

## OAuth Redirect URLs

In Supabase Dashboard → Authentication → URL Configuration:

- Production: `https://your-domain.com/auth/callback`
- Staging: `https://staging.your-domain.com/auth/callback`
- Local: `http://localhost:1000/auth/callback`

## PWA Verification

- Manifest: `/manifest.webmanifest`
- Service worker: `/sw.js` (generated by vite-plugin-pwa)
- Icons: 192x192, 512x512 in `public/`

## Verification Commands

```bash
# Build (requires .env with VITE_ vars or CI secrets)
npm run build

# Environment congruence (local .env.local)
npm run verify:env

# RLS coverage
npm run verify:rls
```
