# Design Direction — uhome

## Core Principle

**Tools don't have to be bland or boring to be serious.**

uhome prioritizes calm clarity, human-friendly language, and responsive interaction over analytical depth or feature overload.

## Design Principles

- **Calm clarity** over analytical depth
- **Expressive minimalism** — not flat or lifeless
- **Neutral power balance** — UI listens, doesn't talk
- **Ambient, glassmorphic surfaces** for depth and tactility

## Visual Language

- Dark mode primary with light mode support
- Warm whites and soft grays (light mode)
- Deep grays and subtle accents (dark mode)
- Single accent hue (sage/slate)
- Status via glow and tint
- Subtle motion (150–250ms)
- Matte/grain texture overlays
- Reflective gradients on cards

## Motion & Interaction

- **Duration**: 150–250ms for most interactions
- **Easing**: ease-out or soft spring
- **Respect reduced-motion** preferences
- **Animate state changes**, not page loads
- **Hover effects**: subtle lift, shadow bloom
- **Number animations**: count up/down on value changes
- **Skeleton loaders** instead of spinners

## Avoid

- Flat, lifeless cards
- Loud gradients
- Over-animation
- Playful or cartoonish elements
- Decorative animations
- Looping or pulsing effects

## Inspiration

- Notion
- iOS system UI
- Linear
- Banking-grade UI (calm, premium)

## Language Standards

- Use **human-friendly language** over accounting jargon
- "This property kept ~82% of its rent" not "Net margin: 82.3%"
- Plain language: "Rent collected", "Upkeep costs", "Net profit"
- Avoid: "Accounts receivable", "Net operating income", "Depreciation"
